## General System Settings

![542 SYSTEM General Pn](/images/542/542-SYSTEM-General.png)

### GPIO CONFIG

- **GPI VOICE CHANGE PRESET**:\
  When GPI is activated, it switches the processing preset.
  - The trigger signal is generated in the Studio when microphones are enabled.
  - It can reach the processor via **hardware** connection or **LAN** (/AoIP option models).
  - The field on the right defines the preset to apply (default: **VOICE-LOUD**).
- **GPO TRIGGER**:\
  List of events that can trigger the GPO output:
  1. AUDIO SILENCE
  2. INPUT FAIL BACKUP
  3. OVERMODULATION
  4. LOW RF POWER
  5. FORCE FM MONO

<Info>
  See [GPIO connection](/es/unidex/instalacion-gpio) details
</Info>

---

### SECURITY

Allows enabling password protection by unchecking **Disable Password Protection** (default: enabled).

- Default password: `1234`
- To modify it, type in the **PASSWORD** field and press **APPLY**
- Rules: up to **8 characters**, only **UPPERCASE** letters and numbers
- **FRONT PANEL CHANGES (LOCK/UNLOCK)**:\
  Locks or enables access from the front panel.\
  If locked, password is required to modify from the front panel.

---

### User Interface

- **LOUDNESS EBU REFERENCE**: Loudness reference level for general monitoring (default: 0 dBfs)
- **AUTO-OPEN LEFT MENU**: Automatically expands the menu when mouse approaches
- **ENABLE TOOLTIPS**: Activates help popup messages over controls and indicators

---

### NETWORK

- Network configuration
- Factory default: **DHCP** mode
- A **static IP** can be assigned by disabling DHCP
- IP address is shown on the unit's front panel or via LAN using **Solidyne Multi-Discovery** tool

<Info>
  For more details see [Ethernet Connection and Access](/es/542/542-instalacion-web)
</Info>

---

### TECHNICAL REPORT

Generates a report with equipment operating parameters and configuration.

- The report is saved to the browser's downloads folder
- Can be sent to **technical support** for diagnostics

---

### Status and Logs

- **SYSTEM TIME**:
  - If the unit has Internet access, it syncs time with an **NTP** server
  - It can also be synchronized from PC using **SYNC FROM PC**
  - Date and time are only used in reports and logs
- **SYSLOG**:
  - Internal memory event log
  - Reserved for **specialized personnel** in diagnostics and technical support
