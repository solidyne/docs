## Audio stream from another console

The AoIP module can be configured to **receive an RTP stream** generated from another Solidyne console with streaming support (e.g.: **D612**, **2600 series**, **Unidex series**, coder/link **ADA102**).\
The encoding device must have the **same firmware (STL)**.

By default, the console comes in **CODER (Studio Encoder)** mode. To receive streams, it must be configured as **DECODER**.

### Receiver Configuration (Decoder)

1. Access the IP module from a web browser (see [IP Module Access](/en/DX816/816-stl)).
2. In the top menu, click **Location**.
3. **Change** the mode to **→** **Transmitter Decoder**.
   - Factory default: **Studio Encoder**, generates unidirectional streaming.
   - As **Decoder**, default values are adopted (only modify IP and ports).
4. Go to **Configuration → Streaming**.
5. In **Port**, enter the port used by the encoder (the transmitting equipment).
6. In **URL**, leave **0.0.0.0**.
7. Click **Apply** to confirm.

### Buffer Adjustment (RTP Latency)

The **input buffer** (expressed in milliseconds) prevents audio dropouts when bandwidth is insufficient:

| Audio Format        | Recommended Buffer                          |
| ------------------- | ------------------------------------------- |
| PCM                 | 40–80 ms                                    |
| Compressed (e.g. MP3) | >80 ms (depends on codec and decoding)      |

<Tip>
  Larger buffer → fewer dropouts, but higher **audio latency**.\  
  Optimal value depends on **bandwidth** and **format** used.
</Tip>

The audio format is always determined by the **transmitter equipment**.

### Decoded Audio Connection

- Audio received via Ethernet is sent to **balanced inputs** using an external cable (RJ45 to RJ45, supplied).
- Connection: **AoIP Output Option → BAL-1 or BAL-2**.
- On DX822 with dual AoIP modules: output branched to both inputs (BAL-1 and BAL-2) using a special "Y" cable (supplied).

## Receiving Broadcasts Using a Computer

It is possible to receive DX/AoIP transmissions on a computer using RTP-compatible software.\
Recommended: **VLC Player** (free – [videolan.org/vlc](http://www.videolan.org/vlc/)).

**VLC Format Configuration:**

- **PCM16 @ 44.1 kHz**, or
- **MP3** in one of its modes.

Other formats may require an **SDP** payload file.

**Procedure in VLC:**

- Menu: **Media → Open Network Stream**.
- In URL field enter:

  `[protocol]://[source_IP]:[port]`

For example:

RTP://0.0.0.0:3030

- `RTP` → protocol.
- `0.0.0.0` → any incoming IP.
- `3030` → port used.

<Note>
  Some VLC versions accept: `RTP://@:3030`
</Note>
