## Ligação Estúdio–Transmissor

- Nos estúdios, a console com opção **/AoIP** opera como **codificador** e gera um streaming de áudio para o transmissor.
- No transmissor, um **decodificador** recebe o fluxo e o envia para o **processador de áudio** (por exemplo, Solidyne **ADA102**).

  Alguns processadores de FM podem receber e decodificar o stream de áudio (por exemplo, Solidyne 542APC /AoIP)

**Requisitos de rede:**

- Estúdio e Transmissor na **mesma LAN** com ligação ponto a ponto (física ou sem fio), ou
- **VPN** sobre Internet de alta performance (consultar com técnico de redes).

## Conexão e acesso

**Procedimento:**

1. Conectar o console à rede usando cabo **Ethernet**.
2. Verificar **LEDs** no conector Ethernet: indicam que os módulos AoIP estão **alimentados por USB** (pelo menos um USB conectado a um PC ligado ou fonte USB externa). Os LEDs devem piscar.
3. A rede atribui **IP por DHCP**. Com o IP obtido, o LED verde do RJ45 fica piscando.\
   Se não houver DHCP, o módulo buscará um **IP livre** (pode levar alguns minutos).
4. Obter o IP com **Solidyne Multi-Discovery**.

  <Note>
     **Solidyne Multi-Discovery**

     Se não houver acesso físico ao console, é possível [**baixar a ferramenta Multi-Discovery**](/es/DX816/816-discovery) que detecta dispositivos Solidyne conectados à rede.
</Note>

5. Abrir um navegador (Firefox/Chrome) e digitar o **IP** encontrado. Aparecerá o **Painel de Controle IP**.

## Configuração de streaming

### **IP de destino (Configurações básicas)**

A tela inicial de status indica a configuração da porta IP. O módulo de fábrica vem configurado como _Codificador: Studio Encoder_ (opção "Location" → 'Studio Encoder').

Para verificar a configuração, acesse a opção **"Configuration"** e no menu à esquerda selecione **"Basic settings"**.

| Parâmetro            | Configuração / Descrição                                   |
| -------------------- | ---------------------------------------------------------- |
| Location (Site type) | De fábrica em **Studio Encoder** (codificador no Estúdio). |
| Stream method        | **Push (RTP)**                                             |
| URL                  | Inserir **IP ou DNS** e **porta de destino** do streaming. |
| Apply                | Pressione para confirmar valores.                          |

| Parâmetro     | Valor de fábrica / Recomendação | Notas                               |

| Input source  | Line Stereo                      | Não modificar.                      |
| Format        | PCM 16 bits stereo               | Padrão.                             |
| Sample rate   | 44.1 kHz                        | Padrão. 48 kHz não suportado.       |

### Tela de status

| Campo                    | Descrição                                                                        |
| ------------------------ | -------------------------------------------------------------------------------- |
| Site type                | Modo de trabalho (ex., **Studio Encoder** em Estúdio).                           |
| Stream mode              | Evento/método configurado para iniciar transmissão para o remoto.                |
| Keep-alive               | Estratégia de manutenção de conexão ativa.                                       |
| Connection status        | Status do link. **Verde** = OK com IP de destino; **Vermelho** = problema de conexão. |
| Incoming Stream          | Status de streaming de entrada (inativo em codificador unidirecional).           |
| Outgoing Stream          | Status de streaming de saída (ativo no codificador).                             |
| Audio input              | Fonte de áudio utilizada (analógicas).                                           |
| Audio format             | Formato aplicado ao áudio transmitido (ex.: PCM16/48k).                          |
| Input/Output audio level | Níveis reais de áudio (dBFS) de entrada/saída.                                   |
| Relay 1 .. 4             | Não implementado.                                                                |
| Remote inputs            | Não implementado.                                                                |
| Local inputs             | Status de entradas locais.                                                       |
| Local relays             | Não implementado.                                                                |

## Link digital por micro-ondas

Para distâncias de **até 45 km**, pode-se utilizar um link de dados ponto a ponto em **5,8 GHz** (ou **2,4 GHz** dependendo do país), padrão **802.11.x**:

- Banda **livre**, sem autorizações especiais.
- Permite transportar áudio **PCM 16 bits / 44,1 kHz** (sem compressão) ou áudio **codificado**.
- Suporta **conexão bidirecional** para aplicações especiais.

## Alimentação de módulos AoIP

- Os módulos AoIP são alimentados por **USB**; exigem pelo menos **um USB** conectado a um **PC ligado**.
- Sem PC via USB, ou com o PC desligado, o serviço IP **não estará disponível**.

<Note>
  Quando não houver PC conectada, pode-se usar uma **fonte de alimentação USB padrão (5 V)** para energizar os módulos AoIP.
</Note>
